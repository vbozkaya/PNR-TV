# Bug Fix: Subtitle "Off" Selection Not Working

Sen uzman bir Android geliştiricisisin. `PlayerViewModel.kt` dosyasında tespit edilen kritik bir mantık hatasını düzeltmeni istiyorum.

## 🐛 Sorun Tanımı
Kullanıcı alt yazıyı "Kapalı" (Off) konuma getirdiğinde, alt yazı kapanmıyor ve varsayılan dile (Auto Select) geri dönüyor.
Log incelemesinde, `selectSubtitleTrack` fonksiyonunda yanlış API kullanıldığı tespit edildi.

**Hatalı Kod:**
Kodda `setRendererDisabled` fonksiyonuna parametre olarak `rendererIndex` yerine `trackType` (C.TRACK_TYPE_TEXT) veriliyor.
ExoPlayer'da `TRACK_TYPE_TEXT` sabiti genellikle `3` değerine sahiptir, ancak Text Renderer genellikle `2.` indistedir. Bu nedenle yanlış renderer hedefleniyor veya hiçbir şey olmuyor.

## 🛠️ Yapılacak Düzeltme

Lütfen `app/src/main/java/com/pnr/tv/ui/player/PlayerViewModel.kt` dosyasını aç ve `selectSubtitleTrack` fonksiyonunu aşağıdaki gibi düzelt:

1.  `if (trackInfo == null)` bloğunun içine git.
2.  Şu satırı bul:
    ```kotlin
    builder.setRendererDisabled(androidx.media3.common.C.TRACK_TYPE_TEXT, true)
    ```
3.  Bu satırı, Track Type tabanlı devre dışı bırakma fonksiyonu ile değiştir:
    ```kotlin
    // Renderer Index yerine Track Type ile işlem yapan doğru metod:
    builder.setTrackTypeDisabled(androidx.media3.common.C.TRACK_TYPE_TEXT, true)
    ```

## 📝 Kontrol Listesi
- Sadece `if (trackInfo == null)` (Kapalı durumu) bloğundaki ilgili satırı değiştir.
- `builder.clearSelectionOverrides(...)` satırını koru (bu gereklidir).
- Diğer mantıkları (else bloğu veya save subtitle işlemleri) değiştirme.

Bu değişiklik, ExoPlayer'ın tüm Text tipindeki renderer'ları devre dışı bırakmasını ve alt yazının gerçekten kapanmasını sağlayacaktır.
